GameCreator引擎并不包含任何游戏玩法逻辑，游戏玩法逻辑需要在项目层中实现。

而游戏模板就是指使用GameCreator引擎实现了玩法以及使用编辑器制作了对应的相关编辑器面板的工程。

打开F11进入脚本编辑器，开始编辑脚本

## 单机版

1.创建全局变量：Game，作为GameBase或其子类的实例。

```ts
var Game: GameBase = new GameBase();
```

2.监听引擎初始化完成：使用EventUtils事件工具类监听引擎初始化完毕事件

```ts
EventUtils.addEventListener(ClientWorld, ClientWorld.EVENT_INITED, Callback.New(function () {
// 编写任意逻辑 
}, this), true);
```

3.新的游戏：调用“新游戏”后系统会派发进入场景的事件，监听进入场景的事件后再编写逻辑：

```ts
// 先监听进入场景的事件：新游戏、读档、切换场景
// @param sceneModelID 场景模型ID
// @param state 状态 0-切换游戏场景 1-新游戏 2-读取存档
EventUtils.addEventListener(ClientScene, ClientScene.EVENT_IN_NEW_SCENE, Callback.New((sceneModelID: number, state: number) => {
// 创建场景、场景对象 等等逻辑
}, this));
// 调用引擎提供的功能：新的游戏
SinglePlayerGame.newGame();
```

4.读取存档：调用引擎提供的功能：“读取存档”，后系统会派发进入场景的事件，监听进入场景的事件后再编写逻辑。（如果已有游戏存档的话则可以读取成功）

```ts
// 先监听进入场景的事件：新游戏、读档、切换场景
// @param sceneModelID 场景模型ID
// @param state 状态 0-切换游戏场景 1-新游戏 2-读取存档
EventUtils.addEventListener(ClientScene, ClientScene.EVENT_IN_NEW_SCENE, Callback.New((sceneModelID: number, state: number) => {
// 创建场景、场景对象 等等逻辑
}, this));
// 调用引擎提供的功能：新的游戏
SinglePlayerGame.loadGame(1, Callback.New((success: boolean) => {
if(!success){alert("读取存档失败");return;}
}, this));
```

通常情况下新的游戏或是读取存档都是通过界面按钮点击后再调用的。

有些休闲小游戏不需要制作新游戏和存读档案，则在引擎初始化完毕后自由编写任意的逻辑

## 网络版

### 客户端

1.创建全局变量：Game，作为GameBase或其子类的实例。

```ts
var Game: GameBase = new GameBase();
```

2.监听引擎初始化完成：使用EventUtils事件工具类监听引擎初始化完毕事件

```ts
EventUtils.addEventListener(ClientWorld, ClientWorld.EVENT_INITED, Callback.New(function () {
// 编写任意逻辑 
}, this), true);
```

3.连接服务器（在编辑器中的通用配置可以设置默认连接的服务器和端口，代码修改则参考[Config.api](?file=007-API-2D网络内核/001-通用/015-Config)）

必须先启动完成服务器，否则会连接失败

```ts
// 生成随机的key和UID以便模拟匿名玩家
Game.playerKey = Math.floor(Math.random() * 20000 + 20) + "_asdasdasd";
// 请求登陆：登录成功后会触发ClientScene.EVENT_IN_NEW_SCENE事件进入到指定的场景
ClientWorld.startLogin(Callback.New(function () {
trace("连接成功，登陆游戏，我的UID=", Game.player.uid);
}, this), Callback.New(function () {
alert("网络错误");
window.location.reload();
}, this));
```

4.当连接成功后系统会派发要求登录场景的事件，当接收到该事件后可以去创建场景

当完成后可以通知服务器，然后服务器与客户端开始执行正常的通信逻辑。

```ts
// sceneModelID 表示需要登录的场景ID
EventUtils.addEventListener(ClientScene, ClientScene.EVENT_IN_NEW_SCENE, Callback.New((sceneModelID: number) => {
// 创建场景的逻辑XXX
// 通信逻辑由项目自定义:如远程调用服务器AA类的BB方法，参数[1,2,3]，服务器需要设置权限允许调用该方法
ClientMsgSender.rpc("AA", "BB",[1,2,3]);
}, this));
```

### 服务端

1.必须存在类 GameServerPlayer，底层会创建该名称的玩家类，所以该类必须存在。

主要方便代码提示和项目层的玩家类逻辑编写。

```ts
/**
 * 项目层玩家类
 * 底层会创建该名称的玩家类，所以该类必须存在。
 * Created by 黑暗之神KDS on 2020-03-03 10:01:38.
 */
class GameServerPlayer extends ServerPlayer {
/**
 * 构造函数
 * @param key 玩家唯一key
 * @param uid 玩家唯一uid
 */
constructor(key: string, uid: number) {
super(key, uid);
}
}
```

2.创建场景绑定类，以便实现相关功能，如：

```ts
class GameServerScene extends ServerScene
```

场景的服务端绑定类可以在编辑器中设定（为了创建场景方便，还可以在设置中预先配置好每个新创建的场景默认的绑定类是什么）：

![alt text](/image/zh_hans/library/3.script/1.init/image.png)
