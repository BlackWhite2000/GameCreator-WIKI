## 说明

玩家登陆后会进入对应的场景，场景由场景线程管理，每个场景线程分别管理多个预设的场景（需要管理的场景才会在该线程中实际创建出来）

创建副本则在当前场景线程中进行创建副本

### createScene

```ts
createScene(dataID : number,  copyMode? : boolean): number :
```

【静态】创建场景，如果是副本则在当前线程下创建副本

副本的生命周期可以由项目层的场景子类实现，比如当没有任何一个玩家在场景上超过30分钟后就销毁副本等等的设定

- #### 参数

  - `dataID` 预设参考ID
  - `copyMode` [可选] 默认值=false 是否副本模式

- #### 返回

  - `number` 返回的场景ID，唯一的，同一个场景副本ID与非副本ID也不一样

### getScene

```ts
getScene(sceneID : number): ServerScene :
```

【静态】获取当前线程中的指定场景，不属于当前线程管理的场景获取不到（除非是从当前场景创建的副本）

目前GC服务器底层默认规则按照 sceneID % SCENE_FIXED_THREAD_COUNT 的规则让固定的场景分给不同的线程管理

- #### 参数

  - `sceneID`

### update

```ts
update(now : number): void :
```

当前场景刷新：帧刷，即每帧执行的函数。服务器设定了每秒刷新60次（如果由于逻辑计算造成卡顿的话，实际可能低于60帧）。

- #### 参数

  - `now` unix时间戳

## addSceneObject

```ts
addSceneObject(so : ServerSceneObject,  addToList? : boolean): void :
```

添加游戏对象：这里并未作任何同步客户端的逻辑，应由项目层实现

不同的项目或模板有不同的效果，所有由项目层实现具体功能。比如MMORPG需要用到AOI算法

- #### 参数

  - so 要添加的游戏对象
  - addToList [可选] 默认值=true 是否是预设NPC，是则不自动计算插入空位置上而是根据预设

## removeSceneObject

```ts
removeSceneObject(so : ServerSceneObject,  removeFromList? : boolean,  force? : boolean): boolean :
```

移除游戏对象：这里并未作任何同步客户端的逻辑，应由项目层实现

不同的项目或模板有不同的效果，所有由项目层实现具体功能。比如MMORPG需要用到AOI算法

- #### 参数

  - `so` 要移除的游戏对象
  - `removeFromList` [可选] 默认值=true 是否从列表中移除
  - `force` [可选] 默认值=false 是否强制移除（不强制的情况下会验证ServerSceneObject是否是列表中的这个，否则只看so.index）

- #### 返回

  - `boolean`
