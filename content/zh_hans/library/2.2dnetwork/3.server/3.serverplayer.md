## 说明

玩家在主线程登录后读取数据然后进入到对应的场景线程

离当玩家开场景会将该场景线程下的玩家数据清理掉

主线程储存所有玩家数据，而支线程储存当前线程下的玩家数据，比如一个服务器有ABCD四个玩家，其中A在线程4，BCD在线程5，

那么主线程一共有4名玩家，线程4有1名，线程5有3名。还有可能就是玩家可能在切换场景线程或者上下线中可能在虚空中，或仅存在于主线程。

主线程的数据不一定是最新的数据，但主线程存档玩家时是最新数据

进入线程场景后会创建新的玩家类并将此前数据复制给新建立的玩家类

## 详情

### EVENT_PLAYER_LOGIN

```ts
EVENT_PLAYER_LOGIN : string;
```

【静态】事件：玩家上线，加载玩家数据后派发事件，仅主线程可获取 callback(player:GameServerPlayer)

登录会读取数据库，读取数据库后派发此事件（此时玩家还在主线程未传输到对应的逻辑场景线程中）

```ts
// 如果是主线程的话
if(ServerThread.threadID==ServerThread.MAIN_THREAD_ID){
// 监听玩家登陆
EventUtils.addEventListener(ServerPlayer,ServerPlayer.EVENT_PLAYER_LOGIN,Callback.New((player:GameServerPlayer)=>{
// to do
},this));
}
```

```ts
EVENT_PLAYER_DISPLACEMENT : string;
```

```ts
if(ServerThread.threadID==ServerThread.MAIN_THREAD_ID){
```


### EVENT_PLAYER_MESSAGE

```ts
EVENT_PLAYER_MESSAGE : string;
```

【静态】事件：接收玩家字符串消息

对应的场景线程可以接收到消息，客户端默认情况下发送到自身所在的场景线程，也可以发送到指定的线程（主线程除外）。callback(player:GameServerPlayer,msg:string)

```ts
// 如果是场景线程的话
if(ServerThread.threadID!=ServerThread.MAIN_THREAD_ID){
// 监听玩家登陆
EventUtils.addEventListener(ServerPlayer,ServerPlayer.EVENT_PLAYER_LOGOUT,Callback.New((player:GameServerPlayer,msg:string)=>{
// to do
trace("接收到来自玩家的消息",msg);
},this));
}
```

### execCommonCommand

```ts
execCommonCommand(commandID : number,  inputMessage : any[]): [CommandTrigger](?file=007-API-2D网络内核/001-通用/014-CommandTrigger) :
```

执行独立的事件库事件，建立一个单独的事件线来执行，不影响其他的事件执行

服务器可调用此方法：以触发者和执行者都是该玩家的新建触发线执行独立事件

如果想要指定触发者和执行者来执行新的事件，参考代码：

```ts
// triggerSo = 触发者场景对象
// executeSo = 执行者场景对象
// fromClient = 是否来自客户端请求，涉及到权限问题不是每个事件都允许客户端调用
var commonEvCmd: CommandPage = ServerWorld.getClientCallWorldEvent(commandID, fromClient);
if (commonEvCmd) {
var trigger = new CommandTrigger(CommandTrigger.COMMAND_MAIN_TYPE_CALL_COMMON_EVENT, commandID, this.scene, triggerSo, true, executeSo);
commonEvCmd.startTriggerEvent(trigger, inputMessage);
return trigger;
}
```

- #### 参数

  - `commandID` 事件库事件的ID
  - `inputMessage` 输入的参数，在事件中可通过trigger.inputMessage

### operationPlayerData

```ts
operationPlayerData(uid : number,  func : Function,  args? : any[],  onFin? : Callback,  useVariables? : boolean): void :
```

【静态】通用的操作玩家数据（非当前线程的玩家或离线玩家也可操作）

由于不确定玩家是否在线或是否在别的线程，可以使用该函数来操作

如果是在其他线程：对应线程执行相关的代码

如果是离线玩家的话：读取并解析该玩家的全部数据并操作完毕后储存

注意：

1. 该操作会有较大的开销，谨慎使用。
2. 线程ServerThread.MAIN_THREAD_ID无法使用该函数

```ts
// 比如操作ID=123的玩家，让该玩家2号变量设置为随机数并返回3号变量数值，此处的rd必须通过参数形式传递过去，因为执行func函数时
// 可能在其他线程中执行，而其他线程中仅执行方法，并不会存在rd这个临时变量，所以通过参数传递安全可靠
// 同时相当于将数据传递到别的线程中，传递数据越多则开销也会越大，尽量使用关键数据传输
trace("6666666666666666")
var rd = MathUtils.rand(10000);
trace("临时变量RD=", rd);
ServerPlayer.operationPlayerData(123, (player: ServerPlayer, rd: number) => {
// 操作变量
trace("操作玩家变量", rd);
player.variable.setVariable(2, rd);
return player.variable.getVariable(3);
}, [rd], Callback.New((returnValue: number) => {
// 返回值来自上面的函数的返回值，所以是该玩家的3号变量，也就是说可以读取其他线程或离线玩家的数据
trace("该玩家的3号变量是", returnValue);
}, this), true);
```

- #### 参数

  - `uid` 玩家ID
  - `func` 操作的方法 func(player:ServerPlayer,...args)
  - `args` [可选] 默认值=[] 执行func方法传递的参数：必须可以JSON化的参数
  - `onFin` [可选] 默认值=null 完成时回调
  - `useVariables` [可选] 默认值=true 是否使用变量，使用的话读写变量才能让离线玩家同步数据库
