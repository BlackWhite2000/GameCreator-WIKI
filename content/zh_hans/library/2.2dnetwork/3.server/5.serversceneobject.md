## 详情

### syncClientData

```ts
syncClientData(syncSelf : boolean): any :
```

同步给客户端的数据：包含场景对象和玩家数据（若是玩家的话）

数据中的自定义属性仅包含设定了“当对象出现时[同步该属性]”的属性（[场景对象模型设定]与[玩家数据设定]中可对属性进行设计）

- #### 参数

  - `syncSelf` 是否同步给自己，根据设定可以筛选哪些属性是可以同步给别的玩家客户端的

- #### 返回

  - `any` 允许同步客户端的数据（公开数据，如果是不公开的则不会传输）

### constructor

```ts
constructor(): (soData? SceneObject, presetCustomAttrs?) :
```

构造函数：创建服务器对象需要通过ServerSceneObject::create来创建

- #### 参数

  - `soData` [可选] 默认值=null 参考SceneObject，根据此数据来创建
  - `presetCustomAttrs` [可选] 默认值=null 参考数据 预设属性，若为空则使用模型数据的默认值
  - `player` [可选] 默认值=null 该对象所属的玩家，若是NPC则该属性为空

### dispose

```ts
dispose(): void :
```

销毁

主动创建或克隆或玩家对象才会销毁

### create

```ts
create(): (soData SceneObject, presetCustomAttrs?) :
```

【静态】创建场景对象

实际根据SceneObject的modelID创建对应的类（场景对象模型有自己的实现类 ServerSceneObject_X）

- #### 参数

  - `soData` 参考SceneObject，根据此数据来创建
  - `presetCustomAttrs` [可选] 默认值=null 参考数据 预设属性，若为空则使用模型数据的默认值
  - `player` [可选] 默认值=null 该对象所属的玩家，若是NPC则该属性为空
  - `isClone` [可选] 默认值=true 是否来自克隆（非预设对象此值一般是true）

- #### 返回

  - `ServerSceneObject` 实际的场景对象类

### clone

```ts
clone(fromSceneID : number,  fromSceneObjectindex : number,  toScene : ServerScene,  isCopy? : boolean,  x? : number,  y? : number): ServerSceneObject :
```

【静态】克隆场景对象

可以从A场景中复制场景对象到B场景中

- #### 参数

  - `fromSceneID` 副本所在的场景ID
  - `fromSceneObjectindex` 副本所在的索引
  - `toScene` [可选] 默认值=null 克隆至的场景（比如当前场景）
  - `isClone` [可选] 默认值=true 是否来自克隆（非预设对象此值一般是true）
  - `x` [可选] 默认值=null 如有则使用x作为默认坐标x，否则使用克隆源的坐标x
  - `y` [可选] 默认值=null 如有则使用y作为默认坐标y，否则使用克隆源的坐标y

- #### 返回

  - `ServerSceneObject` 实际的对象类

### update

```ts
update(now : number): void :
```

更新函数

由子类实现，一般用于场景帧刷调用场景对象的帧刷，以便逐帧执行逻辑

- #### 参数

  - `now` 当前时间 unix时间戳
